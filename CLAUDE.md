定位：每次挑**一篇**计算机 / AI 领域的里程碑论文，把它的核心思想、方法与影响提炼出来、讲透。目标是：**读完这一页，约等于读懂了这篇论文——它解决什么问题、关键 idea 是什么、怎么做的、结果如何、为什么改变了一个领域——读者不必再逐行啃原文和公式。** 很多论文引用上万，其实核心就一两个真正的新想法——你的活就是把那一两个想法拎出来、讲到读者真正 get、能对着面试或工作复述。**不是摘要、不是文献综述、不是导读，是提纯。**

## 1. 写一篇单篇「论文精读」（核心）
写两个文件：中文 `{slug}-paper{N}.html`、英文 `{slug}-paper{N}.en.html`。

**每页两种模式，顶部一个 `.modebar` 药丸切换（默认落在科普版，localStorage 记住选择；见 paper1 的 CSS + 页尾内联切换脚本）：**
- **科普版（`.mode-pop`）**：给零基础读者的**纯类比、零数学、零公式**短版（~600–900 字，5–6 节）。**深度按四层阶梯控制：① 现象（旧世界的痛）→ ② 点子（核心想法）→ ③ 机制直觉（它「怎么」做到——用生活比喻讲，如 QKV=「找什么/我是什么」两张标签、残差=「批注文稿不重抄」）→ ④ 机制精确（公式/复杂度记号/结构细节）。科普版必须讲到 ③、绝不进 ④**（连 `O(n²)` 记号都不出现，只说人话「词一多两两互看、计算量涨得快」）。每个概念要过「饭桌复述测试」：非技术读者听完能转述给第三人；锚不到生活经验的内容属于 ④，挪去精读版。保留**恰好一句**诚实的代价/局限，防营销腔，但别展开。结构参照 paper1/2：干了什么（用读者听过的东西如 ChatGPT 作锚）→ 打个比方/先说个怪事 → 新在哪（对比旧世界）→ 怎么做到的（机制直觉）→ 带来了什么 → 一句话记住。末尾放 `.switch-hint` 链到精读版。这是默认视图。
- **精读版（`.mode-deep`，`hidden`）**：下面这**八节**深读，**分量压在第 4 节**。Glossary 小节放在精读版里。

两种模式都用 `<h2>` 分节（同样的金色标题风）。八节：

1. **一句话** — 这篇论文提出了什么、解决了什么。
2. **坐标（一两句）** — 作者/团队 + 场合/年份 + 它接续谁、又被谁接续；别展开生平、别堆客套。
3. **问题与动机** — 在这篇之前人们怎么做、卡在哪、为什么是真问题。讲清「旧世界的痛」。
4. **核心思想 / 方法，讲透（主体）** — 把真正新的那一两个想法拎出来，每个：**是什么 + 直觉/类比 + 机制（白话）+ 为什么这样设计**。让没读过原文的人也能复述、能用。
5. **关键结果** — 用什么实验/数据证明它 work：数据集、指标、基线、代表性数字。诚实给数字。
6. **为什么重要 / 影响** — 它如何改变领域、催生了什么、成了什么的基石。
7. **局限与批评** — 诚实写局限、后来被发现的问题、有力的反对/超越工作。不吹捧。
8. **要点收尾** — 把整篇最该记住的揉成 6–10 条要点（问题、核心 idea、机制、结果、影响、局限都点到）。

**不写「谁该读 / 怎么复现 / 读完读什么」。**

## 2. 写作要求（重点，和姊妹站不同）
- **篇幅：中文 2500–4000 字，宁精炼勿注水。** IT 论文精读不需要堆字数——把核心机制讲透即可，讲清了就收，别为凑字重复。（比 deep-reading 短。）
- **面向零基础读者（硬规则）**：假设读者**没有该领域基础**。每篇紧接「一句话」之后放一个**独立的 Glossary 小节**（用正式 `<h2>`——中文页 `<h2>Glossary · 术语表</h2>`、英文页 `<h2>Glossary</h2>`，与其它小节并排，别嵌在一句话里；下面接 `.glossary` 虚线框裹一个 `<ul>`，见 paper1），把读这篇**必须先懂、但不是这篇首创**的基础概念（如 RNN、CNN、神经网络、词向量、哈希、TCP、事务、公钥……该文会用到的前置词）**每个用一句大白话讲清**。宁可多花两三行讲基础，也别让新手卡在一个没解释的名词上。正文里其余术语继续随名即释。
- **数学从简、白话讲原理（硬规则）**：**不容易理解的数学推导过程可以省略**，但必须**用大白话把原理/直觉讲清楚**。需要时用行内 `<code>` 给出关键式子（如 `softmax(QKᵀ/√d)V`）并紧跟一句白话解释它在算什么，**不追求完整推导**。判据：读者能不能不看公式就明白「这一步在干嘛、为什么」。
- **配图是硬要求**：凡涉及**架构、数据流、机制的空间结构**（如网络结构、注意力流程、编码器-解码器、共识流程、哈希环、时序），**必须配示意图**。默认用**内联 SVG 自绘**（自包含、跟随本页配色、不依赖外链、可离线）——参照 `attention-is-all-you-need-paper1.html` 里的 `<figure><svg>…</svg><figcaption>…</figcaption></figure>` 写法。每张图配一句中文/英文 `figcaption` 说明。只有当存在**权威且授权明确**的现成图（如 Wikimedia Commons，遵守 hub 的 Commons 规则）时才用外链图，否则一律自绘。英文页的图用英文标注。
- **版式向 system-design 看齐**：无衬线字体（`-apple-system` / SF Pro / Noto Sans SC）、渐变标题、等宽字体（mono）做小标签与元信息、玻璃拟态卡片——**直接复用 paper1 的内联 `<style>` 块**，别退回 deep-reading 的衬线风。
- **配色是本站专属的琥珀/铜色**（gold `#f0b429` → copper `#e8794b` 渐变，冷暗底 `#0f1216`）——**别改成青绿**（那是姊妹站的）。复用 paper1 的 style。
- **真实准确、严禁编造**：不杜撰实验数字/架构细节/引用；把机制和结构讲对——这是技术站的生命线。技术细节拿不准就讲保守。批评/局限诚实写。
- **中英双语，两版都地道**：split 模式，`{slug}-paper{N}.html` + `.en.html` 各自独立成页、langbar 互链。**英文页除 langbar 的「中文 →」外不得出现汉字**（含 SVG 里的标注）。
- **中文页术语首次出现补英文**：`中文术语（English）`，如 自注意力（self-attention）、残差连接（residual connection）。缩写/英文名（Transformer、BLEU、CAP、RSA）照原样。
- **术语随名即释（硬规则）**：引入任何概念/机制/缩写，必须当场一句话说清它**是什么、在算什么**。判据：没读过原文的工程师读完这句能不能复述？不能就补或删。别 name-drop。
- **重点加粗**，克制。

## index 维护：roadmap-first（写论文时把灰色占位转成链接，勿 append 重复）

本仓 index.html / index.en.html 已改为「路线图先出」：清单里还没写的论文已作为灰色占位行 <div class="entry todo">…</div>（无 href、不可点、todo 类）预先列出，顺序随 TOPICS.md 清单。写某篇论文时：按 slug 在两个 index 里找到该论文的灰色占位行，原地改成 <a class="entry" href="{本期文件名}">（去掉 todo 类、加 href，内部结构不变），绝不要在末尾另 append 一行（否则重复）。只有该论文清单里没有对应灰色占位行时才 append。

**index 按主题分组显示**：两个 index 的 `.list` 里，条目按主题簇分组，每组前有一行 `<div class="group">组名<span class="tag">— 副标</span></div>` 小标题（深度学习·现代 AI / Google 分布式系统 / 分布式共识·一致性·存储 / 数据库·数据模型 / 操作系统·网络架构 / 密码学·安全·去中心化 / 万维网·搜索·信息 / 计算与信息理论基石 / 软件工程·编程思想）。**原地转链接天然保持分组**（灰行本就在对的组里），不用动 group 标题。`<!-- entries -->` 标记在「深度学习·现代 AI」组内（已发布论文之后）——仅当某论文在 index 里找不到灰色占位行时才 fallback append 到该标记前，且要 append 进它所属主题组，别破坏分组。新增主题簇时，中英两版同步加一行 `.group` 标题。

**TOPICS 顺序 = 写作优先级；index 分组顺序 = 展示顺序，两者可以不同。** routine **永远从 `TOPICS.md` 顶部往下取第一个未做 slug** 来决定下一篇写什么——只认 TOPICS，不看 index 的分组次序。index 只是「书架 + 路线图」展示：把**已发布的核心主线放前面**、灰色占位按主题就近归组即可，不必逐字镜像 TOPICS 的优先级排序。例：TOPICS 里「深度学习·现代 AI（次优先 · 训练技巧 / 生成 / RL）」这一簇被排到 Google 分布式系统之后（先做分布式），但 index 展示上这几条灰行仍可留在「深度学习·现代 AI」大组里跟已发布论文放一起——因为 routine 靠 slug 找灰行原地转链接，与它显示在哪个组无关。


### 每次运行先「对齐」灰色占位（清单增长时自动补灰行）

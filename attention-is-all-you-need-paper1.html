Attention Is All You Need（Transformer）

Attention Is All You Need · Vaswani 等 · Google · NeurIPS 2017

EN →

一句话

这篇论文提出了 Transformer：一个彻底丢掉循环（RNN）和卷积（CNN）、只靠「注意力」一种机制来处理序列的架构。它让模型一次性看到整句话的所有词、并行地算出每个词该关注谁，于是既训得快、又能一步抓住相距很远的词之间的关系。这个 2017 年为机器翻译而生的架构，后来成了 BERT、GPT 乃至今天几乎所有大模型的共同地基。

坐标

八位作者来自 Google Brain 与 Google Research，论文发在 2017 年的 NeurIPS。在它之前，序列建模的王者是循环神经网络（RNN，recurrent neural network），尤其是 LSTM，通常再配上 2014–2015 年 Bahdanau 等人引入的注意力（attention）作为一个附加组件。这篇论文的激进之处，是把注意力从「配角」直接提成「唯一的主角」——标题 Attention Is All You Need 本身就是一句宣言。它上承 seq2seq（序列到序列）与注意力两条线，下启之后所有预训练大模型。值得一提的是，在当时「去掉循环结构」并不是显而易见的正确选择——RNN 主导序列建模已近三十年，敢把它整个拿掉、赌注意力足以独立支撑，是这篇论文真正的赌性所在。

问题与动机

要理解它的价值，得先看清 RNN 的两个死结。第一是不能并行。RNN 必须一个词一个词地算：第 t 个词的隐状态依赖第 t-1 个的隐状态，像多米诺骨牌一样必须顺次推倒。句子越长，这条链越长，训练越慢，还吃不满 GPU 那种「一次算一大批」的并行算力。

第二是长距离依赖抓不住。一句话里「它」指代前面哪个名词，可能隔了几十个词。在 RNN 里，这份信息要沿着时间步一格一格传过去，路径很长，梯度在长路径上容易消失或爆炸，远处的关联就被稀释掉了。卷积网络（CNN）能并行，但要让相距很远的两个位置「连上」，得堆很多层卷积才够得着。

其实注意力这个机制早在 2014 年就有了——Bahdanau 等人给 seq2seq 翻译加过它：让解码器在生成每个译词时，回头「看一眼」源句里最相关的部分，缓解了长句翻译时信息被压进一个固定向量的瓶颈。但那时注意力只是挂在 RNN 骨架上的一个附加组件，主干还是循环网络，顺序计算的枷锁一点没解开。这篇论文的胆识在于反过来想：既然真正干活的是注意力，那能不能把 RNN 整个扔掉、只留注意力？

于是核心问题浮现：能不能有个架构，既能像 CNN 那样并行计算，又能让句子里任意两个词『一步直连』——不管隔多远，路径长度都是 O(1)？Transformer 的答案是：能，而且只需要注意力。

核心思想 / 方法，逐个讲透

自注意力：每个词自己去问「谁和我相关」

这是全篇的心脏。做法是：句子里每个词都生成三个向量——查询（Query）、键（Key）、值（Value）。你可以这样想象：Query 是「我想找什么」，Key 是「我是什么、能被谁找到」，Value 是「找到我之后，我能提供的实际信息」。这三个向量不是词本身，而是用三个可学习的权重矩阵，把同一个词向量投影出来的三副面孔——训练就是在学「怎么投影，才能让该相关的词相关起来」。

一个词要更新自己的表示时，就拿自己的 Query 去和句子里所有词（包括自己）的 Key 做点积，得到一串「相关度分数」；分数越高，说明那个词和我越相关。把这串分数过一个 softmax（归一化成一组加起来为 1 的权重），再拿这组权重去对所有词的 Value 加权求和——就得到了这个词的新表示。写成公式就是那句著名的：

Attention(Q, K, V) = softmax(QKᵀ / √dₖ) V

白话讲：每个词按「相关度」把全句其他词的信息汇总到自己身上。它的革命性在于——句子里任意两个词，无论相隔多远，都通过这一次矩阵运算直接连上了，路径长度是常数 1，再不用像 RNN 那样沿时间步跋涉。而且这是一次大矩阵乘法，天然可以在 GPU 上并行。

举个经典例子：「The animal didn't cross the street because it was too tired（那只动物没过马路，因为它太累了）」——句中的「它（it）」到底指动物还是马路？人一眼就懂指动物。自注意力就是让「it」这个词的 Query 去和全句每个词的 Key 打分，结果它给「animal」分了很高的权重、给「street」很低——于是模型学会了：更新『it』的表示时，主要从『animal』那里取信息。更妙的是，把原因换成「it was too wide（太宽了）」，同一套机制会让「it」自动改指「street」。这种「一个词动态地决定该听谁的」，正是注意力的威力，也是 RNN 那种固定的顺序传递做不到的。

那个不起眼的 √dₖ（除以键向量维度的平方根）也不能省：当维度较高时，点积的数值会变得很大，softmax 被推进「几乎全 1 或全 0」的饱和区，那里的梯度接近于零、训练几乎学不动。除以 √dₖ 是把分数的方差拉回正常范围，让训练稳定——一个小细节，却是能不能训起来的关键。

多头注意力：同时用好几种「关注视角」

作者没有只做一次注意力，而是做 h 次（原文 h=8）。具体是把 Q、K、V 分别投影到 8 组更低维（每组 64 维）的子空间里，各自独立地做一遍注意力，最后把 8 个结果拼接起来再投影回去。这就是多头注意力（multi-head attention）。

为什么要这样？因为单独一个注意力「头」，只能学会一种关注模式——比如专门盯语法上的相邻词。而语言里的关系是多种多样的：指代关系、句法依存、语义搭配……多个头让模型能在不同的表示子空间里，同时关注不同类型的关系。就像看一幅画，一只眼睛看构图、另一只看色彩，合起来理解更全面。

论文里可视化过这些头，发现它们真的自发分了工：有的头专门把动词连到它的宾语，有的头负责把代词连回它所指代的名词，还有的头关注句子的分隔与边界结构。没人手把手指定谁学什么，这些分工是训练中自己长出来的。

位置编码：给「没有顺序感」的注意力补上词序

纯注意力有个副作用：它天生对词序无感。因为它是对一个集合做加权求和——你把输入的词打乱顺序，算出来的结果只是跟着换个位置，模型分不清「狗咬人」还是「人咬狗」。RNN 靠时间步天然带顺序，卷积靠局部窗口带位置，纯注意力两样都没有。

作者的解法是位置编码（positional encoding）：给每个位置人为造一个向量、加到词向量上，把「你在第几个位置」这个信息直接注入进去。他们用一组不同频率的正弦和余弦函数来生成这些向量。为什么用三角函数？一是不同频率的波形叠加，能让模型从数值里推断出位置；二是相对位置可以用三角恒等式线性地表示出来，方便模型学「相隔 k 个词」这种相对关系；三是理论上能外推到比训练时更长的句子。这个向量是直接加到词向量上的（维度一样，不是拼接），你可以想象成：每个位置都戴上一串由不同频率的波组成的独特「条码」，词向量里从此既有「我是什么词」，也有「我在第几个位置」。

编码器–解码器：两摞积木搭起来

整个 Transformer 是经典的编码器–解码器（encoder–decoder）结构。编码器是 6 个相同的层摞起来，每层两件东西：一个多头自注意力（让源句每个词互相看）、一个逐位置前馈网络。解码器也是 6 层，但每层多一件东西——编码器–解码器注意力，让生成译文时能回头去关注源句的相应部分（这正是早期 Bahdanau 注意力的功能，如今被吸收进统一框架）。

解码器的自注意力还有个关键改动：带掩码（masked）。因为翻译是从左到右一个词一个词生成的，生成第 t 个词时不能偷看第 t+1 个（那是还没生成的答案）。掩码就是在算注意力时把「未来位置」的分数强行压成负无穷，softmax 后权重为零，保证自回归生成不作弊。

为什么要摞 6 层？因为堆叠带来逐层抽象：靠下的层多在处理表层的词与词邻接关系，越往上，注意力越能把这些信息一步步组合成短语、句法乃至语义层面的结构——和 CNN 越深越能从边缘拼出物体、再拼出场景，是同一种直觉。深度在这里换来的是「表示的层次」。

两个不起眼却缺不得的配件：残差 + 层归一化、逐位置前馈网

每个子层外面都套了一层残差连接（residual connection）加层归一化（layer normalization）：残差就是「把输入直接加到输出上」（承接 ResNet 的思想），让梯度有一条捷径回传，几十层也能稳定训练；层归一化则把每层激活的分布拉稳。此外每层还有个逐位置前馈网络（position-wise FFN）——对每个位置独立地过一个两层小网络（512→2048→512 维），给模型补上非线性变换能力。注意力负责「在词之间搬运信息」，FFN 负责「在每个词内部深加工」，两者分工。有个常被忽略的事实：Transformer 里大部分参数其实都躺在这些 FFN 层里（因为中间那层宽达 2048 维），注意力本身反倒不占多少——所以说「注意力就是全部」时，别忘了真正装知识的大仓库是 FFN。

顺带算一笔账：为什么它又快、又抓得远

论文里有张对比表，把三种做法摆在一起看，能一眼看懂 Transformer 好在哪。衡量的两个关键量是：每层的计算量，和「最大路径长度」——即信息要从句子一端传到另一端，最少得经过几步（这个数越小，长距离依赖越好学）。

  • RNN：最大路径长度 O(n)——两个相距 n 个词的位置，信息要一格一格传 n 步，还没法并行。
  • 自注意力：最大路径长度 O(1)——任意两词一步直连；每层计算量约 O(n²·d)（n 是句长，d 是维度），因为要算每一对位置。

关键的权衡在这里：当句子长度 n 小于维度 d 时（翻译里通常如此，句子几十个词、维度 512），自注意力每层的计算量其实和 RNN 相当，却把路径长度从 O(n) 一举压到 O(1)——这就是「又快、又抓得远」的账。代价是计算量对 n 是平方的，句子一长就吃紧——这也埋下了后文「局限」那条 O(n²) 的伏笔。

关键结果

论文在机器翻译上给出硬证据。在 WMT 2014 英译德任务上，大模型拿到 28.4 BLEU（BLEU 是机器翻译质量的标准打分，越高越好），刷新了当时的最好成绩，比之前包括模型集成在内的最强结果还高出 2 个 BLEU 以上；在英译法上达到 41.8 BLEU，同样是单模型新高。

论文给了两档模型：base（约 6500 万参数）用来快速迭代，big 用来冲最高分。更惊人的是成本：那个 big 模型只在 8 张 P100 GPU 上训了 3.5 天，训练开销远低于当时那些动辄训好几周的顶尖模型——既更准，又更省。省从哪来？正是前面那笔账：甩掉了顺序依赖，一整句话可以一次性喂进去并行算，GPU 被喂满，墙上时钟自然大幅缩短。论文还顺手在英语句法分析（constituency parsing）上验证了它能泛化到翻译以外的任务。「更好 + 更快 + 更省」三样一起拿到，是它能迅速被全领域接受的直接原因。

为什么重要 / 影响

这不是一次渐进改良，而是一次范式转变。有意思的是，后来的模型把这套编码器–解码器拆成了三个家族各自发扬：只用编码器的（BERT 一系）擅长理解与分类，用双向注意力读懂整句；只用解码器的（GPT 一系）擅长生成，用带掩码的自注意力一个词一个词往下写；编码器–解码器都要的（T5、翻译模型）擅长「输入一段、输出一段」的转换。「预训练一个大模型、再微调或提示到具体任务」的整个时代，就是从这个架构开始的。

更深远的一点是可并行 → 可规模化。正因为 Transformer 甩掉了 RNN 的顺序枷锁，把模型和数据一路堆到千亿参数才在工程上变得可行，直接铺就了后来「大力出奇迹」的规模化（scaling）之路。它还溢出到语言之外：视觉（Vision Transformer）、语音、蛋白质结构预测（AlphaFold 大量用注意力）、多模态……今天几乎每个前沿 AI 系统的心脏，都是这篇论文里的那块积木。

更深一层看，它带来一种惊人的统一：只要你能把一样东西切成一串「token」（词、图像块、音频片段、氨基酸、甚至一步棋），就能塞进同一个 Transformer 去建模——从前每个领域各有各的专用架构（文本用 RNN、图像用 CNN、语音用另一套），如今被一个通用的序列建模器大面积收编。这种「一套架构通吃」，正是过去几年 AI 能被少数几个团队快速推到前沿的底层原因之一。

局限与批评

  • O(n²) 的复杂度（最该知道的一条）：自注意力要算每一对位置之间的相似度，计算量和显存都随序列长度平方级增长。处理长文档、高分辨率图像就非常吃力。这直接催生了一大批「高效注意力」变体（Longformer、Performer，以及工程上的 FlashAttention 等）来省这笔账。
  • 位置编码是权宜之计：原始的正弦编码外推能力其实有限，后续被相对位置编码、RoPE（旋转位置编码）、ALiBi 等更好的方案逐步替代。
  • 数据与算力饥渴：Transformer 几乎没有 CNN/RNN 那种内建的归纳偏置（inductive bias，即对局部性、平移不变等先验假设），好处是灵活、上限高，代价是要用更多数据才学得动；在小数据集上未必赢过卷积。一个典型例证：把 Transformer 搬到视觉的 ViT，要在 JFT-300M 这种上亿张图的超大数据集上预训练，才明显超过同级的卷积网络；数据不够时，缺少归纳偏置反而成了短板。
  • 标题其实夸张了：「注意力就是你所需的全部」是句响亮的标语——但正如上面所讲，前馈层、残差、层归一化、位置编码一个都不能少，纯注意力本身并不够。这点后来常被同行善意调侃。
  • 「注意力权重=可解释性」被质疑：早期有人把注意力权重当成「模型在看哪里」的解释，后续研究指出注意力权重并不等于特征重要性，拿它当解释要小心。

十句话精华

① Transformer 的一句话：丢掉 RNN 和卷积，只用注意力处理序列，既能并行训练、又能让任意两个词一步直连。

② 它要解决的痛：RNN 必须顺次计算、无法并行，且长距离依赖要沿时间步长途传递、容易丢失。

③ 自注意力的机制：每个词生成 Query/Key/Value，用 softmax(QKᵀ/√dₖ)V 按相关度把全句信息汇总到自己身上，任意两词路径长度为 1。

④ 那个 √dₖ 不是装饰：它把点积方差拉回正常，防止 softmax 落进梯度接近零的饱和区、训不动。

⑤ 多头注意力让模型在多个子空间同时关注不同类型的关系（指代、句法、语义），一个头只能学一种模式。

⑥ 纯注意力对词序无感，于是用正弦/余弦位置编码把「第几个位置」注入进去；解码器的自注意力还要加掩码，防止生成时偷看未来。

⑦ 残差 + 层归一化让几十层稳定可训，逐位置前馈网负责在每个词内部做非线性深加工——注意力搬运信息，FFN 深加工信息。

⑧ 结果是「更好 + 更快 + 更省」：英译德 28.4 BLEU 刷新 SOTA，8 卡 3.5 天训完，成本远低于同期模型。

⑨ 影响是范式级的：BERT、GPT 全建在它上面，可并行带来可规模化，还溢出到视觉、语音、蛋白质、多模态——当代 AI 的共同地基。

⑩ 局限要诚实记住：注意力是 O(n²) 的、长序列吃力；位置编码是权宜之计；它数据饥渴；标题也夸张了——前馈、残差、归一化、位置编码一个都不能少，纯注意力并不够。
